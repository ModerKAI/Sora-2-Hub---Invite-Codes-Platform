Лендинг с экраном загрузки

Структура проекта:

- css/style.css — стили
- js/main.js — логика экрана загрузки
- assets/videos/ — папка для видео загрузочного экрана
- index.html — основной файл

Куда положить видео для экрана загрузки

1) Положите файлы в папку `assets/videos` внутри проекта `landing`.
   Рекомендуемые имена:
   - `loader.webm` — WebM версия (рекомендуется для современных браузеров)
   - `loader.mp4` — MP4 (H.264) версия как запасной вариант

2) Форматы и рекомендации
   - Форматы: WebM (VP9/AV1) и MP4 (H.264). WebM даёт лучший сжатие, MP4 — совместимость.
   - Разрешение: 720p (1280x720) обычно достаточно. Для качественного эффекта можно использовать 1080p, но учтите вес файла.
   - Длительность: 3–8 секунд обычно достаточно для интро. Длинные видео замедляют загрузку.
   - Компрессия: используйте ffmpeg для перекодирования и битаудит 2500–4000 kbps для 720p.
     Пример команды:
     ffmpeg -i input.mov -vcodec libx264 -crf 23 -preset medium -acodec aac -b:a 128k loader.mp4

3) Оптимизация и хостинг
   - Не добавляйте большие видео в git без LFS. Лучше хранить видео на CDN или объектном хранилище и указывать внешние ссылки в `index.html`.
   - Для разработки можно положить файлы в `assets/videos` и открыть `index.html` через локальный сервер.

Как запустить локально (рекомендуется использовать локальный сервер):

- Если установлен Python (рекомендуется): откройте PowerShell в папке `d:\sora_2\landing` и выполните:

```powershell
python -m http.server 8000
```

и откройте в браузере `http://localhost:8000`.

- Или используйте расширение VSCode Live Server.

Примечания

- В `index.html` видео подключено через `<source>`: сначала `loader.webm`, затем `loader.mp4`.
- Скрипт `js/main.js` скрывает экран загрузки после окончания видео, по нажатию кнопки "Пропустить" или через запасной таймаут (10s).

Если хотите, могу:
- добавить миниатюру/poster для видео;
- подключить preloader и прогресс-бар загрузки данных;
- или сделать адаптивную разметку и анимации для первого экрана.
